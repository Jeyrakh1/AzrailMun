# Защита APK koyo.apk от декомпиляции

## Реализованные меры защиты

### 1. Обфускация Smali
- **Пакет**: `com.xzaqp5om.vou6y1dd` → `a.b.c.d`
- **Классы**: Все 1026+ классов приложения переименованы в случайные идентификаторы (a, b, c, x7k2m9 и т.д.)
- **Структура**: Плоская структура `a/b/c/d/` вместо иерархической

### 2. Скрытие исходного кода
- Удалены все директивы `.source "SourceFile"` — скрыты имена исходных файлов
- Удалены директивы `.line` — скрыта привязка к номерам строк

### 3. Анти-отладка
- Проверка `Debug.isDebuggerConnected()` — завершение приложения при подключении отладчика
- Инжектировано в `onCreate` главной Activity

### 4. Подпись
- APK подписан для распространения

## Использование

```bash
# Полная сборка
./apk_protect/build.sh

# Или пошагово:
apktool d коуо.apk -o koyo_decompiled -f
python3 apk_protect/protect.py
apktool b koyo_protected -o koyo.apk --use-aapt2
jarsigner -keystore koyo.keystore koyo.apk koyo  # storepass: koyopass
```

## Результат

Защищённый APK: **koyo.apk**

При декомпиляции злоумышленник увидит:
- Бессмысленные имена классов (a, b, c, x7k2m9...)
- Скрытый пакет a.b.c.d
- Отсутствие информации об исходных файлах
- Затруднённую отладку
